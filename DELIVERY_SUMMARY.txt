HOW TO USE THE NEW COMPONENTS
===============================================================================

1. ENTITYTABLE COMPONENT
   The main workhorse for all entity pages. Replaces manual table building.
   
   Usage:
   <EntityTable
     columns={[
       { key: 'name', label: 'Name' },
       { key: 'status', label: 'Status', render: (s) => <StatusBadge status={s?.name} /> },
     ]}
     data={systems}
     entityName="System"
     onView={(item) => router.push(`/dashboard/systems/${item.id}`)}
     onDelete={(item) => deleteSystem(item.id)}
     getMaintenanceLogs={getEntityMaintenanceLogs}
   />

2. ENTITY MINI DASHBOARD
   Shows status breakdown for filtering.
   
   Usage:
   <EntityMiniDashboard
     totalCount={systems.length}
     totalIcon={Zap}
     statusCounts={[
       { status: 'Active', count: 5, icon: CheckCircle, color: 'green' },
       { status: 'Inactive', count: 2, icon: AlertTriangle, color: 'red' },
     ]}
     filterPath={(status) => `/dashboard/systems?status=${status}`}
   />

3. STATUSBADGE COMPONENT
   Use for all status displays (already exists, now used consistently).
   
   Usage:
   <StatusBadge status={item.status?.name} />

4. MAINTENANCE TABLE
   Embedded in EntityTable, but can be used standalone.
   
   Usage:
   <MaintenanceTable
     logs={logs}
     onEdit={(log) => setEditingLog(log)}
     onDelete={(logId) => deleteLog(logId)}
   />
